# 公開手順（Step 1: 静的サイト公開）

費用ゼロでの公開手順。**オーナー作業**と**ジョブズ作業**を分けて記載。
（方針: アカウント作成・認証はオーナー / コード準備・push・検証はジョブズが承認後に実行）

## 1. GitHubアカウントとリポジトリ（オーナー・約10分）

1. https://github.com でアカウント作成（無料）
2. 「New repository」→ 名前 `fx-backtester`、**Public** を選択、READMEは追加しない
3. このPCでGitHubの認証を通す（いずれか）:
   - GitHub Desktop をインストールしてログイン、または
   - `gh auth login`（GitHub CLI）

## 2. コードのpush（ジョブズ・オーナー承認後）

```
git remote add origin https://github.com/<ユーザー名>/fx-backtester.git
git push -u origin main
```

## 3. GitHub Pages有効化（オーナー・約2分）

1. リポジトリの Settings → Pages
2. Source: 「Deploy from a branch」/ Branch: `main` / フォルダ: `/ (root)` → Save
3. 数分後に `https://<ユーザー名>.github.io/fx-backtester/` で公開される

## 4. 公開後すぐやること

- [ ] スマホ・PC両方で表示確認（ジョブズがチェックリストで検証）
- [ ] **匿名アクセス解析**: Cloudflare Web Analytics（無料・Cookie不使用・個人情報非収集）をオーナーが登録 → 発行されるビーコン1行をジョブズが index.html に追加
- [ ] **フィードバック導線**: Googleフォームをオーナーが作成（項目案: 使った感想 / 欲しい機能 / 不具合）→ URLをジョブズがフッターに挿入
- [ ] 紹介投稿（X等）: 文面はジョブズが作成、**投稿はオーナーが実行**（SNS投稿ルール準拠）

## 注意事項

- **HistDataのデータファイルは同梱・再配布しない**（再配布は先方規約に抵触しうる）。ユーザーには自分でダウンロードして読み込んでもらう（README参照）
- 投資助言に当たる機能（個別の売買推奨・シグナル配信）は実装しない。ツールはあくまで検証環境の提供に徹する
- Step 2（ユーザーデータ収集・広告掲載）は需要が確認できてから。その際は利用規約・プライバシーポリシー・同意UIを整備してから着手する
